Garp Features:
Powering Innovation

Garp Blockchain is built to power the next generation of decentralized finance, tokenized assets, and enterprise-grade solutions.

Hybrid
Consensus

POS DPoS PoH

Garp uses a hybrid consensus (PoS, DPoS, PoH) for high throughput and security[cite: 14]. Validators are selected via stake-weighted randomization (PoS) for energy efficiency and security , and elected by the community (DPoS) to ensure decentralization and fairness. Proof of History (PoH) provides cryptographic proof of transaction ordering, allowing for faster transaction finality.

Scalability

Sharding zk-Rollups Optimistic Rollups 65,000+ TPS

Garp implements dynamic sharding, scaling with network demand to handle 65,000+ transactions per second. To further increase throughput, Garp integrates Layer 2 solutions including zk-Rollups and Optimistic Rollups without compromising security.

WASM Contracts

Rust Go C++ Gas Metering

Smart contracts are powered by a WebAssembly (WASM) VM, providing near-native performance[cite: 34]. Garp supports contracts written in Rust, C++, and Go[cite: 35]. All contracts are optimized with Gas Metering to ensure predictable transaction costs and faster execution.

Developer
Friendly

SDKs CLI Web3.js ethers.js JSON-RPC

Garp provides robust tools for fast development cycles [cite: 18], including SDKs for Rust, Go, and C++[cite: 101, 102, 103]. Integrate your DApp frontend easily with support for libraries like Web3.js and ethers.js[cite: 112, 113]. A powerful CLI and JSON-RPC API are available for deployment and interaction[cite: 8, 23].
